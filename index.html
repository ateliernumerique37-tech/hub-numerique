Projets numériques accessibles

Applications, jeux, outils vocaux et services web conçus pour rendre le numérique plus simple, plus utile et plus accessible, en particulier pour les personnes déficientes visuelles.

Créés ou pilotés par Saif Ayedi, avec une attention forte portée aux lecteurs d’écran, à l’autonomie et aux usages associatifs concrets.

Un écosystème utile, pas une simple vitrine

Ce hub présente une série de projets numériques construits autour d’un même fil rouge : l’accessibilité réelle. L’objectif n’est pas seulement de créer des interfaces qui “passent” techniquement avec un lecteur d’écran, mais des outils réellement confortables, compréhensibles et utilisables dans la vie quotidienne.

Les projets couvrent plusieurs besoins : gestion associative, jeux accessibles, apprentissage, consultation vocale des transports et accès simplifié à des ressources pratiques.

Application associative

H2VL Connect

Application interne de gestion pour l’association H2VL : adhérents, événements, inscriptions, cotisations, communications e-mail, rôles administrateurs, statistiques et tâches d’équipe.

Gestion associative accessible, structurée et adaptée au terrain.

Jeu accessible

Audio Memo

Jeu de mémoire sonore en PWA, conçu en priorité pour les personnes non-voyantes et malvoyantes. Le son, les annonces et le lecteur d’écran sont au cœur de l’expérience.

Jouer avec les sons, pas avec les yeux.

Assistant vocal

Skill Alexa et API permettant de demander vocalement les prochains passages du réseau Fil Bleu à Tours. Le service privilégie des réponses courtes, utiles et adaptées à un usage mains libres.

Méthode

L’approche

Accessibilité d’abord

Les interfaces sont pensées pour les lecteurs d’écran, la navigation clavier, les annonces vocales, les contrastes et les usages sans repère visuel.

Utilité concrète

Chaque projet part d’un besoin réel : gérer une association, jouer, se déplacer, apprendre, chercher une information ou préparer un atelier numérique.

Sobriété technique

Les choix techniques privilégient la robustesse et la maintenance : PWA, Firebase, GitHub Pages, VPS léger, documentation Markdown et architecture compréhensible.

Vision technique

Les projets utilisent des technologies modernes mais raisonnables : Next.js, Firebase, GitHub Pages, HTML/CSS/JavaScript, TypeScript, API Node.js et intégrations vocales. L’idée est de construire des outils solides, documentés et évolutifs, sans complexité inutile.

La documentation Markdown occupe une place importante : elle permet de transmettre le contexte à un humain, à Claude Code, à Cursor, à Codex ou à tout autre agent de développement.
